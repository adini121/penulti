\abstract{\vspace{1em} % Add a gap in the Contents, for aesthetics

In recent years, many software environments have started automating their web application regression tests using tools such as Selenium. However, as the application continues to evolve, existing Selenium tests might not achieve the same behavioral coverage as they do for the version the tests are originally written for. Such Selenium tests can be deemed to be non-robust. The robustness of Selenium tests decreases over time due to factors such as changed functionality, modified GUI elements, etc. Such tests can be less effective in covering the existing functional specifications while maintaining them requires additional efforts. Currently there is no technique to analyze the robustness of Selenium tests to facilitate their assessment and maintenance. 

This Master’s thesis evaluates the robustness of Selenium tests over the version history of web applications. The approach consists of employing Selenium tests to capture the application's behavior as a finite-state model and to detect GUI level differences for different versions of a web application. This thesis establishes \textit{robustness grade} as a quantitative measurement for the robustness. In addition, a set of metrics are derived to establish which factors affect the robustness of Selenium tests the most.
